> アプローチ: 親DD（管理用）。実装は子DDで行う。

## 目的

Excel運用中のカレンダー（業務予定表＋ガントチャート）をWebアプリ化するための、最初の環境構築〜ざっくり動くモック完成までを管理する。

## 背景・課題

- 現状はExcel 2シート構成（シート1=入力用テーブル、シート2=ガント表示）で運用
- シート1の日付→シート2の色付け、を数式で同期しているが、双方向にしたい
- シート2のセル内テキスト（「洗浄」など、その日に何をするかの作業内容）はシート2固有の情報で、シート1には戻らない
- ブラウザ上でガントをドラッグ・リサイズできるようにしたい
- PCのみ対応でよい
- 想定: 社内少人数運用

## 検討内容

### 技術選定

[c:\repo\nanairoware3](c:\repo\nanairoware3) の構成を踏襲しつつ、DBはSQLiteとする。

| 層 | 技術 | 理由 |
|---|---|---|
| BE | Hono + TypeScript | nanairoware3 と同構成 |
| ORM | Prisma | 同上。将来PostgreSQL/MySQL移行も容易 |
| DB | **SQLite** (`prisma/dev.db`) | ファイル1個で完結、Docker不要 |
| FE | React 19 + Vite + TypeScript | nanairoware3 と同構成 |
| バリデーション | Zod | サーバ/フロント共有 |

### DBスキーマ方針

プロトタイプ段階は **JSON型カラム** で動的に持つ。UIが固まったらカラム化する2段階方式。

```prisma
model Job {
  id        Int      @id @default(autoincrement())
  rowNo     Int      @unique
  data      Json     // {category, comment, steps:[{name, startDate, endDate, dailyNotes:{}}]}
  updatedAt DateTime @updatedAt
}
```
