### Prismaスキーマ（初期）

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

model Job {
  id        Int      @id @default(autoincrement())
  rowNo     Int      @unique
  data      String   // JSON文字列（SQLiteはJson型を文字列で保存）
  updatedAt DateTime @updatedAt
}
```

> 補足: Prisma の `Json` 型は SQLite では実態 TEXT。`String` で持って、サーバ側で `JSON.parse / JSON.stringify` ＋ Zodで型検証する方式が一番素直。

### dev起動方法

| コマンド | 内容 |
|---------|------|
| `npm run dev` | Hono サーバ起動（`tsx watch src/index.ts`、port 3000） |
| `npm run dev:fe` | Vite 起動（port 5173、API は proxy で 3000 へ） |
| `npm run db:migrate` | `prisma migrate dev` |
| `npm run db:seed` | シードデータ投入 |

## 決定事項

- 上記ディレクトリ構成で作成
- Prisma スキーマは Job 1テーブル、`data` は String(JSON文字列)
- BE: port 3000、FE: port 5173（Viteのproxyで `/api` → 3000）
- 動作確認は `GET /api/health` と `GET /api/jobs` の応答で行う
