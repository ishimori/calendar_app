## 目的

シート2を **Googleカレンダー的なオブジェクト指向UI** にリデザインする。具体的には:

1. **(必須) ガント上の要素を「掴んで動かす／伸ばす」オブジェクトとして扱う** — 現状の「平面テーブル＋バーオーバーレイ」を、行・バー・メモの3レイヤをそれぞれドラッグ可能にする
2. **メモを「セル内テキスト」から「期間オブジェクト」に格上げ**:
   - 複数日にまたがる（1セルでも2セル以上でも可）
   - 長文可（複数行・スクロール可能）
   - 任意の色を設定可能
3. **バー（工程）自体にも色を任意設定可能**
4. **シート1↔シート2 の双方向同期は既存どおり維持**（同一 `Job.data` への PUT で自動同期）

## 背景・課題

### ユーザー要件追加（2026-05-21）

DD-001-3 初版起票直後にユーザーから以下の要件が追加された:

- メモは現状の短文だけでなく **もう少し長いメモも書きたい**
- メモの **色を任意に変えたい**
- 1セルだけでなく **2セル以上をくっつけて1つのメモ** にしたい
- (必須) 現在の平面的なセル描画をやめ **Googleカレンダー的なオブジェクト操作UI** にしたい

### これがなぜ全面書き直しか

現データモデル [src/lib/schemas.ts:6-11](../../src/lib/schemas.ts#L6-L11) は `dailyNotes: Record<"YYYY-MM-DD", string>` で「1日=1テキスト」を前提にしている。
新要件「複数日にまたがる・長文・色付き」は **メモを期間オブジェクト化** することと同義で、データモデルから刷新が必要。同時にUIも「セル内入力」から「オブジェクト配置」に変わる。

## 検討内容

### データモデル変更案

```typescript
// 現状（[src/lib/schemas.ts](../../src/lib/schemas.ts)）
Step { name, startDate, endDate, dailyNotes: { [YYYY-MM-DD]: string } }

// 新案
type Color = string  // "#3b82f6" 等（プリセット＋自由入力）

interface Note {
  id: string           // crypto.randomUUID()
  startDate: string    // "YYYY-MM-DD"
  endDate: string      // 単日なら startDate と同じ
  text: string         // 改行を含む長文可
  color?: Color
}

interface Step {
  name: string
  startDate: string
  endDate: string
  color?: Color
  notes: Note[]        // dailyNotes を廃止
}
```

影響範囲: [src/lib/schemas.ts](../../src/lib/schemas.ts), [src/client/pages/GanttPage.tsx](../../src/client/pages/GanttPage.tsx), [src/client/pages/SheetPage.tsx](../../src/client/pages/SheetPage.tsx), [scripts/seed.ts](../../scripts/seed.ts)
モックフェーズのため後方互換は不要（DB リセット可）。

### UI 構造（3レイヤ）

```
┌──── 行（背景セル: 日付グリッド、薄い罫線のみ）────┐
│  ┌── バー（工程の期間、色付き、ドラッグ/リサイズ可）──┐ │
│  │  ┌─Note─┐    ┌────Note (複数日)────┐         │ │
│  │  │ 洗浄 │    │ 検査記録... (長文)   │         │ │
│  │  └──────┘    └──────────────────────┘         │ │
│  └────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

操作仕様（案）:

| 操作 | 対象 | 挙動 |
|---|---|---|
| バー本体ドラッグ | バー | 工程を移動 |
| バー両端ドラッグ | バー | 工程を伸縮 |
| バー上の空き領域ダブルクリック | バー | その日にNote新規作成（単日） |
| Note本体ドラッグ | Note | Noteを移動（バー内で完結、バー外には出ない） |
| Note両端ドラッグ | Note | Noteを伸縮（複数日化） |
| Noteクリック | Note | ポップオーバーで text/color 編集 |
| Note右クリック | Note | 削除メニュー |
| 行の空き領域ダブルクリック | 行 | （将来）バー新規作成 — 本DDではスコープ外 |

既存実装で動作中（両ページとも `PUT /api/jobs/:rowNo` → `invalidateQueries(['jobs'])` で反対側ビューが自動更新）。

### スコープ外

- 月またぎ・複数月ビュー（DD-001-2 から継続: 2026-06-01〜2026-06-30 固定）

- 楽観ロック・複数ユーザ同時編集
- Note のドラッグ&ドロップで別工程に移動
- リッチテキスト（Markdown/書式）— プレーンテキスト改行のみ

## 決定事項

DD-001-4（React モック）のレビューで以下に確定（2026-05-21）:

- [x] **色の選択方式**: プリセット6色 ＋ 自由カラーピッカー（Note・バー共通）
- [x] **Note 編集UI**: クリックでポップオーバー展開（バー外に浮かぶカード、長文 textarea）
- [x] **Note 同士の重なり**: 禁止（ドラッグ・リサイズ時にバリデーションで弾く）
- [x] **バーの色**: ユーザー個別設定（Note 編集ポップオーバー内から変更）

その他の確定事項:

- **操作仕様**: バー本体ドラッグ＝移動、両端ハンドル（6px）＝リサイズ。Note も同様。クリック/ドラッグは 5px 閾値で分離
- **Note 新規作成**: バー上の空き領域をダブルクリック → その日に単日 Note 作成
- **データキー**: Note は `crypto.randomUUID()` でクライアント生成
- **本実装スコープ（DD-001-5）**: モック実装をそのまま `src/client/pages/GanttPage.tsx` に反映 + Zod schema 更新 + 既存 `dailyNotes` を `notes[]` に移行（モック段階なので `db:reset` 前提）
