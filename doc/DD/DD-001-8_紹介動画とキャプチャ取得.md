## 目的

DD-001 ファミリーの締めくくりとして、本プロジェクトのアプリ動作を一通り見せる紹介動画を撮影し、主要画面のキャプチャを取得する。新規メンバーや関係者がリポジトリを開かなくても、`doc/DD/DD-001-8/` を覗けば「どんなアプリか」がわかる状態にする。

## 背景・課題

- DD-001-7 で画面リデザインが完了したが、静止スクショだけでは「触ったときの操作感」が伝わらない
- 既存の Playwright MCP は録画機能を持たないため、Playwright ライブラリを直接スクリプトで叩いて動画化する

## 検討内容

### 録画手段

| 候補 | 採否 | 理由 |
|---|---|---|
| Playwright MCP の組み合わせ | ❌ | 録画 API なし。スクショ繋ぎでは間延びする |
| Playwright (`recordVideo` オプション) | ✅ | 1スクリプトで動画＋スクショ。webm 形式で 1〜2MB に収まる |
| OS の画面録画ツール (手動) | ❌ | 再現性なし、CI に乗らない |

### スクリプトのシナリオ（30秒前後）

1. シート1 オープン → タンクグループ表示
2. ガントに切替 → 全体像
3. 工程バーをドラッグ移動
4. タンクフィルタを開いて T-02 を除外 → T-01/T-03 のみ表示
5. T-02 を戻す
6. マイルストーン (◆顧客レビュー) をダブルクリック → カスタムダイアログ
7. ダイアログで名前変更して保存
8. 出荷予定イベントをドラッグして日付変更
9. 工程バーをダブルクリック → 詳細編集モーダル
10. モーダルを閉じる → シート1 に戻る

各ステップ間に 800〜1500ms のウェイトを挟み、視聴者が追える速度にする。

### 環境前提

- 撮影前に `db:reset && db:seed` でクリーン状態にしておく
- 撮影後にも再シードして、デモ操作の永続化結果を残さない

## 決定事項

- Playwright を **devDependency** として追加（録画用、E2E テスト基盤としても今後使う想定）
- Chromium のみインストール（FF/Webkit は不要）
- 解像度 1600x900（DD-001-7 のスクショと統一）
- 動画は webm（Playwright デフォルト）。ファイルサイズ重視
- スクリプト: `scripts/record-intro.ts`
- 出力先: `doc/DD/DD-001-8/`

## スコープ外

- mp4 変換（webm のままでブラウザ表示・GitHub プレビュー可）
- ナレーション・字幕（無音）
- E2E テスト整備（本DDは「紹介動画の取得」が主目的）
